The DeepL API provides programmatic access to DeepL’s language AI technology, making it possible to bring high quality translation capabilities directly to your websites and applications.

## Common use cases for the DeepL API

* **Embedding translation in a product you sell:** [Six official SDKs](/docs/getting-started/client-libraries) — Python, PHP, Node.js, .NET, Ruby, Java. Per-customer configuration scales as your user base grows. [Voice API](/api-reference/voice) adds speech-to-text to the same endpoint if audio input is part of your product roadmap.  
  
* **Real-time data processing:** News wires, financial data feeds, content monitoring pipelines. The API handles high character volumes and returns output directly into downstream models or review queues — no batching workarounds required.  
  
* **Automating document workflows:** Integrates into existing document workflows without manual steps or post-translation reformatting. Replaces multi-day agency turnaround with a programmatic process that runs on demand, at the volume your operation requires.  
   
* **Replacing LLM-based translation in production:** LLMs are non-deterministic — same input can produce different output across calls. For RAG systems, AI agents, or any pipeline where downstream processing depends on consistent output, that's a reliability problem. DeepL returns identical output for identical input in most cases.  
   
* **Building CAT tool integrations:** If your product is a CAT tool, [the API fits directly into MT suggestion and review workflows](https://support.deepl.com/hc/en-us/articles/360019358599-About-DeepL-for-CAT-tools). Translation suggestions surface within the translator's existing environment — no context switching, no separate interface. The same REST endpoint that powers automated pipelines works equally well as a human-in-the-loop MT engine.  

If you would like to develop a DeepL plugin for your CAT tool, [please contact us](https://support.deepl.com/hc/en-us/requests/new).


## Why the DeepL API?

- **High-quality text, voice, and document translations:** DeepL [consistently outperforms the competition](https://www.deepl.com/quality.html) in translation quality—and not only for text translation. The API also supports [.docx, .pptx, .xlsx, .txt, PDF, and HTML](/api-reference/document) files.
- **Maximum data security:** For DeepL API paid plans, texts aren’t saved on persistent storage and aren’t used to train our models. And DeepL adheres strictly to EU data protection laws and ISO 27001. [Learn more about data security at DeepL](https://www.deepl.com/pro-data-security/).
- **Customization with glossaries, style rules, and translation memories:** [Specify your own translations for words and phrases](/api-reference/multilingual-glossaries), and customize your translations consistently and at scale.

Try out the DeepL API by [signing up for a free account](https://www.deepl.com/pro-api#api-pricing). For more advanced use cases, [DeepL API paid plans](https://www.deepl.com/pro-api#api-pricing) allow unlimited translation with usage-based pricing, maximum data security, and prioritized execution of translation requests.

### Intended Purpose of the DeepL API

DeepL API is intended to translate or otherwise process general documents or other content provided by the Customer in accordance with the documentation. DeepL API is not intended for any high-risk applications as defined in [Article 6 of the EU AI Act](https://artificialintelligenceact.eu/article/6/) (including any applicable delegated acts adopted by the European Commission on the basis of this provision).
