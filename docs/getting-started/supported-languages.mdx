<Info>
  Please note that Hebrew, Thai and Vietnamese have not been added to the `/languages` endpoint as they are only available in the text translation API using next-gen models. When these languages are available for all translation endpoints (including document translation) they will be added to the `/languages` endpoint and this note will be removed.
</Info>

<Info>
If trying to convert between variants of the same language (e.g. `EN-US` to `EN-GB`), please use the [/write/rephrase](/api-reference/improve-text) API.
</Info>

### Beta languages

DeepL offers additional languages that are currently in Beta phase. These languages have limitations regarding feature support and translation quality.

<Info>
  Beta languages can be enabled by adding `enable_beta_languages: true` to the body of your translation request. Note that these languages do not support glossaries or formality yet.

  Any request with the `enable_beta_languages` parameter enabled will use `quality_optimized` models. Requests combining `enable_beta_languages: true` and `model_type: latency_optimized` will be rejected.

  During the Beta phase, characters translated from and into these languages are not billed and counted against your character threshold. Requests with the beta flag enabled that do not include beta languages are billed normally.
</Info>

<Info>
  As of April 2025, supported languages for [text improvement](/api-reference/improve-text) (DeepL API for Write) have not yet been added to the `/languages` endpoint; we'll be extending the `/languages` endpoint in the near future to include this information.
</Info>

  For the `/write/rephase` endpoint, `writing_style` and `tone` currently work in `DE`, `EN-GB`, and `EN-US`.
</Info>
