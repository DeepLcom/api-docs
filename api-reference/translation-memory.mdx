Translation memories store and reuse previously created translations. Upload your translation memories as TMX files and use them in your text translations to improve consistency and efficiency across your projects.

<Info>
Translation memories are currently uploaded via the DeepL UI. Support for uploading translation memories via the API and other CRUD APIs will follow.
</Info>

<Info>
  The Translation Memory API is currently available only to Pro API subscribers.
</Info>

## About translation memories

A translation memory is a collection of **segments** — paired units of source text and their corresponding translations. When you translate text with DeepL, the API compares your source text against the segments in a translation memory. If a segment is similar enough to your source text (based on the `translation_memory_threshold`), the stored translation is used.

Each translation memory has a single source language and one or more target languages. The supported source languages are: `de`, `en`, `es`, `fr`, `it`, `ja`, `ko`, and `zh`. The supported target languages are: `de`, `en`, `en-gb`, `en-us`, `es`, `es-419`, `fr`, `it`, `ja`, `ko`, `zh`, and `zh-hans`.

### Language variant matching

Only target languages can be variants. A translation memory's target language is compatible with both the base language and its own variant when specified as the `target_lang` in a translation request. For example:

- A translation memory with target language `en-us` can be used when `target_lang` is `en` or `en-us`.
- A translation memory with target language `en` can be used when `target_lang` is `en`, `en-us`, or `en-gb`.

However, if the `target_lang` in the translation request is a **different** variant than the one stored in the translation memory, it will not match. For example, a translation memory with target language `en-us` will **not** work when `target_lang` is `en-gb`. A `404` error will be returned in this case.

## Using translation memories in translation requests

To use a translation memory when translating text, include the `translation_memory_id` parameter in your request to the [text translation endpoint](/api-reference/translate/request-translation). The `source_lang` parameter is not required — language auto-detection still works when using a translation memory. You can optionally set `translation_memory_threshold` (0-100) to control the minimum matching percentage required for a translation memory segment to be applied. The default threshold is **75%**, which we recommend as a minimum.

<Info>
  Any request with the `translation_memory_id` parameter enabled will default to use the `quality_optimized` model type. Requests combining `translation_memory_id` and `model_type: latency_optimized` will be rejected.
</Info>

<Tabs>
<Tab title="cURL">
```sh Example request: translate with a translation memory
curl -X POST 'https://api.deepl.com/v2/translate' \
--header 'Authorization: DeepL-Auth-Key [yourAuthKey]' \
--header 'Content-Type: application/json' \
--data '{
  "text": [
    "The agreement shall be governed by the laws of the Federal Republic of Germany."
  ],
  "target_lang": "DE",
  "translation_memory_id": "a74d88fb-ed2a-4943-a664-a4512398b994"
}'
```

```sh Example request: translate with a custom threshold
curl -X POST 'https://api.deepl.com/v2/translate' \
--header 'Authorization: DeepL-Auth-Key [yourAuthKey]' \
--header 'Content-Type: application/json' \
--data '{
  "text": [
    "The agreement shall be governed by the laws of the Federal Republic of Germany."
  ],
  "target_lang": "DE",
  "translation_memory_id": "a74d88fb-ed2a-4943-a664-a4512398b994",
  "translation_memory_threshold": 85
}'
```

```json Example response
}
```

## List translation memories

Retrieve a list of translation memories associated with your account.

<Tab title="cURL">

```sh Example request: list translation memories

--header 'Authorization: DeepL-Auth-Key [yourAuthKey]'
```

## Query parameters

<ParamField query="page" type="int">
Zero-based page index. Default: 0. Use with `page_size` to get the next page of translation memories.
</ParamField>
<ParamField query="page_size" type="int">
The maximum number of translation memories to return. Default: 10. Minimum: 1. Maximum: 25.
</ParamField>
